# µPS reference drafts (Photoshop)

Reference PSD and Adobe ground-truth PNGs for implementing and tuning **gulp-mu-ps**: compositor features (layer effects, blend modes, layer links, text), both **ButtonAndIconCreator** modes, and a single **overview contact sheet** for visual comparison.

## Files

| Path | Role |
| :--- | :--- |
| `../drafts/buttons.psd` | Legacy sample (copy from `µCSS 1.0 (Photoshop)/µCSS/examples/drafts/`) |
| `../drafts/mups-reference.psd` | Comprehensive reference (generated by JSX) |
| `../../tools/photoshop/build-mups-reference.jsx` | Builds `mups-reference.psd` |
| `../../tools/photoshop/export-mups-reference.jsx` | Exports PNG references to `out/` |
| `../../tools/build-reference-overview.mjs` | Stitches `out/` into `out/overview/mups-reference-sheet.png` |
| `../../tools/photoshop/MuPsReferenceLib.jsx` | Shared helpers (CpFX/PaFX, shapes, text, layout builders) |
| `../../tools/photoshop/MuPsLayerEffects.jsx` | Layer-style presets and parameter variants |

## PSD structure (`mups-reference.psd`)

Extends the known-good **ButtonAndIconCreator** tree from `buttons.psd`:

```
layouts/
  aqua/, alu/                 — full button layouts (normal, hover, …)
  fx/                         — isolated layer effects, multiple PS settings each
    solidFill, solidFill_multiply
    dropShadow, dropShadow_soft, dropShadow_hard
    innerShadow, innerShadow_aqua, innerShadow_multiply, innerShadow_wide
    innerGlow, innerGlow_tight, innerGlow_warm
    outerGlow, outerGlow_soft, outerGlow_white
    gradientOverlay, gradientOverlay_diag, gradientOverlay_reverse
    bevelInner, bevelInner_chisel, bevelPillow, bevelOuter, bevelEmboss
    …/normal/{ bgrd, icon }
  stacks/                     — combined styles copied from buttons.psd
    stack_aquaIcon, stack_aquaBgrd, stack_aluIcon
  blend/                      — aqua icon style + varied blend modes
    blend_normal, blend_multiply, blend_screen, …
  links/
    normal/ { icon_master, follower }   — PS layer link (move together)
icons/
  but_*                       — original glyphs
  glyph_disc, glyph_group, glyph_text, glyph_text_light
topsets/                      — CreateByTopLayerSets demo (frame_01 … frame_03)
```

**Not in this PSD** (separate legacy references under `oldsrcs/…/examples/`): **SpriteAtlas**, **TileSheet**, **SequenceStrip** / DSD strips, **AppIconMaker**, **cursors.psd**. Those pipelines consume PNG output; this PSD covers the **PSD compositor** surface.

## Workflow

### 1. Prepare sample PSD

```text
copy "µCSS 1.0 (Photoshop)\µCSS\examples\drafts\buttons.psd" gulp-mu-ps\examples\drafts\
```

### 2. Build reference PSD (Photoshop)

**Troubleshooting (no debugger needed):** run **`test-mups-script.jsx`** first (same folder). It shows script paths and whether `buttons.psd` was found.

**File → Scripts → Browse…** → `gulp-mu-ps/tools/photoshop/build-mups-reference.jsx`

Creates `examples/drafts/mups-reference.psd`. The script saves automatically (alert **Saved:** …). You do **not** need File → Save manually.

If nothing appears on disk, Photoshop usually aborted before `saveAs` — read the error alert. Common causes: `buttons.psd` missing, or helper scripts not loaded when using **Browse** (fixed in current JSX via `$.evalFile`). If the **mups-reference** document stays open after an error, save it manually to `examples/drafts/mups-reference.psd`.

### 3. Export Adobe reference PNGs

**File → Scripts → Browse…** → `gulp-mu-ps/tools/photoshop/export-mups-reference.jsx`

Exports **one PNG per test case at PSD master resolution** (no 1×/@2x pairs — sufficient for visual regression). Progress is written to `examples/drafts/mups-reference-export.log` after each file.

**Re-runs** skip PNGs that already exist under `out/` (no re-render). To force a full re-export, set `MuPsExportSkipExisting = false` at the top of `export-mups-reference.jsx` before running the script.

Output under `examples/reference/out/`:

| Folder | Content |
| :--- | :--- |
| `bc/aqua/`, `bc/alu/` | **ButtonAndIconCreator.Create** — full icon × state matrix |
| `fx/` | Isolated effects with style transfer onto `glyph_disc` |
| `flat/fx/` | Direct composite of each fx layout (no glyph transfer) |
| `stacks/` | Combined stacks from real `buttons.psd` layers |
| `blend/` | Blend-mode matrix on aqua background |
| `topsets/` | **ButtonAndIconCreator.CreateByTopLayerSets** frames |
| `links/` | Layer-link demo |
| `text/` | Text glyphs with aqua icon style transferred |
| `compositing/chart/` | Known color patches + rainbow bar (`chart.png`) |
| `compositing/fillOpacity/` | Rainbow disc + bevel at fill 100 / 50 / 0 % |
| `compositing/layerOpacity/` | Rainbow disc at layer opacity 100 / 50 % |
| `compositing/layerAlpha/` | Rainbow disc with opaque vs 50 % **pixel** alpha |

### Opacity model (µPS vs Photoshop)

Three independent controls — do not confuse them:

| Control | PSD source | Effect in µPS |
| :--- | :--- | :--- |
| **Pixel alpha** | Alpha channel per pixel in `imageData` | Shape mask for effects + compositing alpha |
| **Fill opacity** | `fillOpacity` (iOpa), 0…1 | Scales **fill alpha only**; layer styles stay full strength |
| **Layer opacity** | `opacity`, 0…1 | Scales **entire** layer (fill + styles) at composite time |

**Rainbow discs** (not flat gray/red) expose wrong RGB scaling, channel swaps, and blend bugs that solid fills hide.

Synthetic checks without Adobe: `test/Compositing.test.mjs`. After changing this chart: rebuild PSD → export → `ReferenceRender` tests for `compositing/*`.

### 4. Build overview contact sheet (Node)

```bash
node gulp-mu-ps/tools/build-reference-overview.mjs
```

Writes `examples/reference/out/overview/mups-reference-sheet.png` — one labelled image covering all exported sections.

### 5. Compare with µPS

Render µPS output (same layout as `out/`):

```bash
node gulp-mu-ps/tools/render-reference.mjs
node gulp-mu-ps/tools/compare-images.mjs gulp-mu-ps/examples/reference/out/stacks/ gulp-mu-ps/test/.tmp-mups-reference/stacks/
```

Automated regression (MAE tolerances per section):

```bash
cd gulp-mu-ps && npx mocha test/**/*.test.mjs --grep ReferenceRenderer
```

Manual inspection:

```bash
node gulp-mu-ps/tools/inspect-psd.mjs gulp-mu-ps/examples/drafts/mups-reference.psd
node gulp-mu-ps/tools/compare-images.mjs gulp-mu-ps/examples/reference/out/fx/ gulp-mu-ps/test/.tmp-mups-reference/fx/
```

Use `ButtonAndIconCreator.Create()` / `.CreateByTopLayerSets()` against `mups-reference.psd` and compare with `out/bc/` and `out/topsets/`. Pass **`retina: false`** so µPS writes the same resolution as the Adobe reference PNGs (no paired 1×/@2x needed for these tests).

## Effect variants (tuning guide)

Isolated `fx/` layouts use **synthetic** single- or multi-parameter presets in `MuPsLayerEffects.jsx`. Combined **`stacks/`** layouts copy the **full layer style** from `buttons.psd` (best ground truth for production-like output). When tuning `Effects.mjs`, prefer:

1. **`stacks/stack_aquaIcon`** — full aqua icon stack (inner shadow, bevel, …)
2. **`innerShadow_aqua`** — isolated inner shadow matching aqua defaults (color burn, spread 14)
3. **Variant pairs** (`dropShadow` vs `dropShadow_soft`, `bevelInner` vs `bevelInner_chisel`) — parameter sensitivity

Re-run **build → export → overview** after editing presets or PSD structure.

## Notes for µPS development

- **Ebenenverknüpfungen:** Legacy µCSS uses **Copy/Paste Layer Style** (`CpFX` / `PaFX`), not PS position links. Both are covered: exports use CpFX/PaFX; `links/` demonstrates PS **layer link**.
- **Blend modes:** Each `blend/*` layout sets the icon placeholder’s blend mode in the PSD. Export copies **layer style + blend mode** onto `glyph_disc` over the aqua **bgrd** (placeholder hidden).
- **Text layers:** Keep live text in the PSD for ag-psd development; rasterize only for ad-hoc PNG checks.
- **Fülloptionen:** Isolated layouts map to `Effects.mjs` (`solidFill`, `gradientOverlay`, `innerGlow`, `outerGlow`, `innerShadow`, `dropShadow`, `bevel`).
- **Fill opacity / layer opacity / pixel alpha:** `ag-psd` reads `fillOpacity` (iOpa). µPS implements the three-way split (see table above). Rainbow discs in `compositing/` expose channel and blend bugs that solid fills hide. Synthetic tests: `test/Compositing.test.mjs`. Rebuild PSD + re-export after changing `build-mups-reference.jsx`.
- Commit updated PNGs under `examples/reference/out/` when adding automated regression tests.

# µPS-Referenz-Entwürfe (Photoshop)

Referenz-PSD und Adobe-PNGs für die **gesamte Compositor-Oberfläche** von **gulp-mu-ps** (Fülloptionen inkl. Varianten, Blend-Modi, Ebenenverknüpfungen, Text, beide ButtonAndIconCreator-Modi) plus **Übersichts-Kontaktblatt**.

Ablauf: `buttons.psd` kopieren → `build-mups-reference.jsx` → `export-mups-reference.jsx` → `build-reference-overview.mjs` → Vergleich mit `compare-images.mjs` / `inspect-psd.mjs` (µPS mit `retina: false`). **Export-Wiederholungen** überspringen vorhandene PNGs in `out/`; Vollständig neu: `MuPsExportSkipExisting = false` in `export-mups-reference.jsx`.

**Deckkraft:** Drei getrennte Stellschrauben (siehe Tabelle oben): **Pixel-Alpha** im Ebeneninhalt, **Füll-Deckkraft** (`fillOpacity`, nur Fill), **Ebenen-Deckkraft** (`opacity`, gesamte Ebene). Regenbogen-Discs statt Einfarbig — erkennt Kanal-/Blend-Fehler besser. Algorithmisch in `Compositor.mjs`; Adobe-PNGs für `compositing/*` nach PSD-Neubau exportieren.

**SpriteAtlas**, **TileSheet**, **SequenceStrip** und **AppIconMaker** haben eigene Legacy-Referenzen unter `oldsrcs/…/examples/` — sie hängen nicht an dieser PSD, weil sie PNG-/Sequenz-Pipelines sind, nicht Layer-Compositing.
